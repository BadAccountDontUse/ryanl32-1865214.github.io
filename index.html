A Git is a version control system that facilitates teams working on the same code at once.

GitHub is the most popular Git. It's a "hub" where you can store open sourced code and collaborate on ideas both big and small.

GitHub a great tool to share your projects with others (job recruiters, friends, family, etc.). The Web Site you built for Info 101 starting today can become your portfolio. Also, if you go on to take more Informatics classes, participate in hackathons, or work on independent projects, the GitHub skills you learn will really help you!

Workshop: Embedding media

In this workshop you will learn how to embed stuff like video and audio in your web page

We'll walk you through some of the standard embeds and then show you how to find out more.

To do

Go through this workshop with your TA to receive XP. Then use what you learned to do your own original work.

Embedding Media with Tags

Some embeds are simply HTML tags that tell the browser to put something on the page

  • You have already seen how to embed an image using the <img> tag. Take the inline CSS workshop to see how to size your images
  • In HTML 5 (the newest version) there is an <audio> tag that lets you embed a little audio player in your page Learn how to use it at W3Schools
  • Similarly there is an HTML 5 <video> tag that lets you embed a video on your page. Go to W3Schools to learn more
  • Flash and other similar stuff that the browser does not know how to render uses an <object> tag
iFrames

The <iFrame> tag inserts someone else's web page right inside yours. Go to W3Schools to learn more

iFrames let you embed all sorts of stuff in your web page. While iFrames used to be frowned upon (stuff goes in and out of style on the web), they are now quickly becoming the method of choice for embedding complicated stuff

  • Embedding Youtube videos is a snap. You just go to the video and copy and past the iFrame code. Learn about the other embedding options at youtube.com
  • Google maps, news and lots of other stuff is quickly embedable by copy and paste.
  • Getting something to happen is easy, getting just what you want can be harder

Here is the heart of the Google Map embed:

"https://maps.google.com/maps?f=q&source=s_q&hl=en&geocode=&q=software+ companies&aq=0&oq=software&sll=47.670993,-122.344752&sspn=0.263556,0.676346& ie=UTF8&hq=software+companies&hnear=&t=m&ll=47.670993,-122.344752&spn=0.263556,0.676346 &output=embed

Looks imposing, no? If you divide it up a little you can start to figure out how it all works

This is the web page that creates the map:

https://maps.google.com/maps?

Here are all the things you can tell the page that creates the map. Some are not so hard to read. The line q=software+companies, for example, is probably where I told the map to show the location of software companies. Most are impossible to figure out without a long explanation. If you cared enough, you could find that explanation on the web and become a Google Map embedding expert!

  • f=q&
  • source=s_q&
  • hl=en&
  • geocode=&
  • q=software+companies&
  • aq=0&
  • oq=software&
  • sll=47.670993,-122.344752&
  • sspn=0.263556,0.676346&
  • ie=UTF8&
  • hq=software+companies&
  • hnear=&
  • t=m&
  • ll=47.670993,-122.344752&
  • spn=0.263556,0.676346&
  • output=embed
